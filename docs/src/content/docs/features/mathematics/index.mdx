# Mathematics

Mercury supports mathematical notation using $\LaTeX$.

## Inline Math

You can write inline math by wrapping it with `$...$`.

```mdx
This is an example of an inline formula: $E=mc^2$.
```

## Block Math

You can write block math by wrapping it with `$$...$$`.

```mdx
$$
\int_a^b f(x) \,dx = F(b) - F(a)
$$
```

## Supported $\LaTeX$ Commands

Mercury uses $\KaTeX$ to render mathematical equations.
For a list of supported $\LaTeX$ commands, please see the following page:

https://katex.org/docs/supported.html

## Options

Mercury uses `remark-math` and `rehype-katex` for math rendering.

Here's how you can configure their options.

### remark-math

You can pass options to `remark-math` using the `mdx.remarbkMath` property.

```ts title="vite.config.ts" {11-13}
import mercury from "@mercurymd/vite-plugin";
import tailwindcss from "@tailwindcss/vite";
import react from "@vitejs/plugin-react";
import { defineConfig } from "vite";

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    mercury({
      mdx: {
        remarkMath: {
          // write your options here...
        },
      },
    }),
    react({ include: /\.(mdx|js|jsx|ts|tsx)$/ }),
    tailwindcss(),
  ],
});
```

For available options, please refer to the following page:

https://github.com/remarkjs/remark-math

### rehype-katex

You can pass options to `rehype-katex` using the `mdx.rehypeKatex` property.

```ts title="vite.config.ts" {11-13}
import mercury from "@mercurymd/vite-plugin";
import tailwindcss from "@tailwindcss/vite";
import react from "@vitejs/plugin-react";
import { defineConfig } from "vite";

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    mercury({
      mdx: {
        rehypeKatex: {
          // write your options here...
        }
      },
    }),
    react({ include: /\.(mdx|js|jsx|ts|tsx)$/ }),
    tailwindcss(),
  ],
});
```

For available options, please refer to the following pages:

https://github.com/remarkjs/remark-math/tree/main/packages/rehype-katex

https://katex.org/docs/options.html
